# Techniques to Task Cross-Reference (BABOK Appendix B)

The formal pairings BABOK documents between its 50 techniques and 30 tasks. Use this when:
- A user is on a specific task and you want to recommend techniques (Section A)
- A user mentions a technique and you want to know where BABOK formally uses it (Section B)

This is a **shortlist**, not a hard rule. BABOK explicitly says techniques can be added, modified, or invented. The pairings below are the ones BABOK documents — they're a strong starting point, not the only fit. When you recommend, pick 2–4 from the formally-paired list, then briefly justify why each fits the user's specific situation.

## Patterns worth noting

- **Most-paired techniques**: Interviews (21 tasks), Workshops (19 tasks), Survey/Questionnaire (16), Risk Analysis & Management (14), Process Modelling (13). These are the workhorses — fluency with them pays back across BABOK.
- **Single-task techniques**: Data Dictionary, Glossary, Sequence Diagrams, State Modelling are all paired only with 7.1 Specify and Model Requirements. Useful but narrow.
- **Technique-richest task**: 6.1 Analyze Current State formally pairs with 26 of the 50 techniques. That's a clue about how broad current-state analysis is — there's no single right way to do it.
- **Technique-leanest tasks**: 4.4 Communicate Business Analysis Information (3 techniques), 4.5 Manage Stakeholder Collaboration (4), 5.5 Approve Requirements (5). These are tightly scoped, mostly governance/coordination.

When recommending, the most-paired techniques are safe defaults but rarely the most insightful. Reaching for a less-common technique that fits the situation specifically is often where the value is.
